## Changelog

### Data publication improvements

- "Open Data" has been redefined as **Public Data**, supporting a broader range of licensing scenarios.
- Added support for **DataCite** and **OpenAIRE** metadata formats across GUI, APIs, and OAI-PMH endpoints.
- Enhanced JSON metadata visibility and handling in file-detail views and workflows.

---

### OneS3 advances (beta)

The **OneS3 service** continues to mature:

- Deployment during cluster setup or into existing clusters.
- Configurable ports and full management via GUI or REST API.
- Improved TLS certificate handling and configuration consistency.

---

### Web GUI performance

- Infinite scrolling, batching, and caching across list views, side-panels, and token tables.
- Better progress indicators and loading feedback across all long operations.
- Performance optimisations for large memberships, group hierarchies, and space views.

---

### Security, resilience, and observability

- Removal of unsafe TLS ciphers with flexible per-service cipher configuration.
- Backpressure mechanisms preventing request flooding between Onezone and Oneprovider.
- Richer structured error reporting and improved diagnostics across all components.
- Numerous fixes improving reliability across storage connectors, networking layers, and quota handling.
